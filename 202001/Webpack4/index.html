插件 plugins

常用插件

  • mini-css-extract-plugin 打包 CSS
  • optimize-css-assets-webpack-plugin 优化 CSS
  • terser-webpack-plugin 打包 JS
  • html-webpack-plugin 打包 HTML
  • clean-webpack-plugin 打包前删除已经打包文件
  • copy-webpack-plugin 复制插件
  • webpack.bannerPlugin 版权插件
  • webpack.IgnorePlugin 忽略模块插件
  • webpack.DllPlugin 打包成动态库
  • webpack.DefinePlugin 定义环境变量
  • webpack.HotModuleReplacementPlugin 热跟新插件
  • webpack.ProvidePlugin 版权标注

模块 module

常用 loader

  • autpprefixer 处理浏览器内核 CSS 前缀
  • css-loader 处理 CSS 中的@import
  • style-loader CSS 加入到 head 标签中
  • postcss-loader CSS loader
  • expose-loader 暴露全局 loader
  • eslint-loader JS 代码校验
  • babel-loader ES6 相关
    • @babel/core 核心
    • @babel/preset-env ES6-->ES5
    • @babel/plugin-proposal-class-properties ES6 class 语法
    • @babel/plugin-proposal-decorators ES6 decorators 语法
    • @babel/plugin-transform-runtime 开发时需要
    • @babel/runtime 上线时需要
    • @babel/polyfill 解决 es6 'foobar'.includes('foo')

配置文件合并

开发的时候可能需要看看发布后的效果，就需要使用发布的配置文件，开发 mode 的发布 mode 下很多配置是一样的，就可能使用一种类似继承的方式
